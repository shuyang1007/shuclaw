# Memory Skill - 长期记忆系统

## 概述

这个 Skill 让 AI 助手（树大虾）能够记住长期信息，跨越不同的对话会话。不再是每次重启就"失忆"，而是像人类一样有持续的记忆。

## 核心能力

### 1. 自动记忆提取
每次对话结束后，AI 会自动分析：
- 用户做了什么重要决定？
- 有什么项目状态需要记住？
- 用户偏好有什么变化？
- 下次对话需要带什么上下文？

### 2. 主动回忆
用户可以说：
- "上次我们说的那个项目怎么样了？"
- "我之前让你记住的事情呢？"
- "继续我们昨天的话题"

AI 会自动搜索记忆库，找回相关上下文。

### 3. 启动记忆加载
每次新会话开始时，AI 自动加载：
- 最近的重要决策
- 正在进行的项目状态
- 用户的长期偏好
- 相关的历史对话摘要

## 使用方法

### 对于 AI（OpenClaw/Kimi）

**存储记忆：**
```python
from memory_system import MemorySystem

mem = MemorySystem()
mem.store_memory(
    content="用户决定用 React + Node.js 开发新项目，预计 2 个月完成",
    memory_type="decision",
    importance=5,
    tags=["项目", "技术栈", "时间线"]
)
```

**检索记忆：**
```python
# 用户问"上次说的项目"
results = mem.recall_memory("项目 React 开发", limit=3)
# 返回相关记忆列表
```

**加载会话上下文：**
```python
# 新会话开始时
context = mem.load_context_for_session()
# 返回格式化的记忆文本，插入到系统提示中
```

### 对于用户

**显式让 AI 记住：**
- "记住我喜欢用 Tab 而不是空格"
- "把这个重要决定记下来：我们要做 X"
- "别忘了，我下周要出差"

**询问历史：**
- "上次我们聊到哪儿了？"
- "我之前让你查的资料呢？"
- "我有什么待办事项？"

**管理记忆：**
- "我有多少条记忆？" → 显示统计
- "清理旧记忆" → 删除不重要或过期的记忆

## 记忆类型

- `fact` - 事实性知识（用户的公司、职位等）
- `decision` - 重要决策（技术选型、方案确定等）
- `preference` - 用户偏好（代码风格、沟通方式等）
- `task` - 待办事项（截止日期、跟进事项等）
- `project` - 项目状态（进度、里程碑等）
- `auto` - 自动提取（默认）

## 技术细节

### 存储结构
```
~/.openclaw/memory/
├── short_term.json     # 短期记忆（低重要性、临时）
├── long_term.json      # 长期记忆（高重要性、永久）
└── notes/              # 长文本笔记
    └── mem_xxxxx.md
```

### 检索算法
1. 关键词匹配（提取中文/英文关键词）
2. 内容包含度（查询词在记忆中的出现）
3. 重要性加权
4. 时间衰减（越新的记忆分数越高）
5. 访问频率（经常被回忆的记忆优先）

### 记忆生命周期
- 短期记忆：30 天后自动清理（除非重要或被频繁访问）
- 长期记忆：永久保存
- 支持手动删除和更新

## 集成到 OpenClaw

### 1. 安装
```bash
cp -r skills/memory-skill ~/.openclaw/skills/
pip install -r requirements.txt  # 如果有依赖
```

### 2. 配置 OpenClaw
在 OpenClaw 的系统提示中添加：
```
你有长期记忆能力。每次对话：
1. 启动时调用 memory_system 加载相关上下文
2. 对话中自动提取重要信息存储为记忆
3. 用户询问历史时调用 recall_memory 检索
```

### 3. 自动记忆工作流
```
用户发送消息
    ↓
AI 处理消息
    ↓
AI 判断是否有值得记住的信息
    ↓
如果有 → store_memory()
    ↓
AI 生成回复
    ↓
（可选）询问用户"需要我记住这个吗？"
```

## 示例场景

**场景 1：项目跟踪**
```
用户：我们要开始一个新项目，用 Python + FastAPI
AI：好的！我将记住这个项目的技术栈。 [自动 store_memory]
...
一周后
用户：那个 Python 项目怎么样了？
AI：让我回忆一下... [recall_memory("Python 项目")]
上次我们决定用 Python + FastAPI，目前进度是...
```

**场景 2：个人偏好**
```
用户：我习惯用空格缩进，不用 Tab
AI：记住了！你偏好空格缩进。 [store_memory, type="preference"]
...
一个月后写代码时
AI：生成代码时自动使用空格缩进（因为我知道你喜欢这样）
```

**场景 3：待办提醒**
```
用户：下周三我有个重要会议，别忘了提醒我
AI：已记录！下周三提醒你开会。 [store_memory, type="task"]
...
下周三对话时
AI：先提醒一下，你今天要参加那个重要会议！
```

## 注意事项

1. **隐私**：记忆存储在本地，不会上传到云端
2. **准确性**：重要记忆建议让用户确认后再存储
3. **数量**：避免存储太多无用信息，定期清理
4. **敏感信息**：密码、密钥等不要存储

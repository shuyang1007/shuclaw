# 葛军式命题方法论
## 从数学命题到AI技能设计的通用框架

---

## 一、核心哲学：好的题目/技能是怎么诞生的？

### 1. 源于真实，高于真实
- **葛军**：不考偏题怪题，考的是数学思想的真实应用
- **迁移**：不炫技，解决真实用户的真实痛点

### 2. 螺旋上升的设计
```
基础层 → 综合层 → 创新层 → 思想层
```

### 3. 反套路思维
- 不是"你会什么我考什么"
- 而是"你没想到但我引导你想"

---

## 二、四步命题/设计法

### Step 1: 挖掘真问题（选题）
**葛军做法：**
- 不看已有题库，回到数学本质
- 问：学生学完这章，应该真正理解什么？

**AI技能设计迁移：**
```
真问题检测清单：
□ 这是用户的痛点还是痒点？
□ 没有这个技能，用户现在怎么解决的？有多痛苦？
□ 解决后，用户的生活/工作会有什么具体改变？
□ 这个需求3年后还存在吗？
```

**例子对比：**
- ❌ 错题：做"小红书自动发布"（炫技导向）
- ✅ 对题：做"帮助IP主理人每天节省1小时内容运营时间"（问题导向）

---

### Step 2: 设计认知阶梯（ scaffolding ）
**葛军做法：**
一道好题不是一步到位，而是：
```
入口宽 → 路径多 → 顶峰陡 → 思想深
```

**AI技能设计迁移：**
```
MVP阶梯：
Level 1: 能跑起来（核心功能）
Level 2: 能容错（异常处理）  
Level 3: 能优化（智能决策）
Level 4: 能共生（主动服务）
```

**小红书技能例子：**
- L1: 能发一条笔记 ✓
- L2: 能处理验证码/登录失效 ✓
- L3: 能根据数据自动优化发布时间/内容策略（待做）
- L4: 能主动提醒"今天热度话题X，建议发一条"（未来）

---

### Step 3: 埋设思维锚点
**葛军做法：**
题目里藏"暗线"，引导学生不自觉地用到核心思想。

**AI技能设计迁移：**
在交互中植入"认知锚点"：

```python
# 不好的设计
用户：发小红书
AI：好的，标题是什么？内容是什么？...

# 葛军式设计（有锚点）
用户：发小红书
AI：🎯 今天的目标是？
   选项：A.测试功能 B.日常运营 C.追热点 D.深度长文
   
   [用户选B]
   
AI：📊 看了你过去10条数据，
   晚8点发布互动率高30%，建议今晚8点发。
   要我设置定时吗？
```

**锚点设计原则：**
1. **不强迫**：提供选项，而非填空
2. **有根据**：每个建议都有数据/逻辑支撑
3. **可跳过**：高级用户可以直接跳过引导

---

### Step 4: 制造"啊哈"时刻
**葛军做法：**
好题目让学生做到某一步时，突然顿悟："原来考的是这个！"

**AI技能设计迁移：**
设计"超预期"的惊喜时刻：

| 普通设计 | 葛军式设计 |
|---------|-----------|
| 按指令发完小红书 | 发完后自动总结："今天比上周同期多花5分钟，主要卡在封面选择，建议下次直接复用模板" |
| 单纯回复评论 | 扫描完评论后说："发现3个人问工具链接，建议置顶一条'工具在主页'的回复" |
| 被动等待指令 | 主动推送："你关注的Polymarket事件有新进展，要发一条分析吗？" |

## 三、避坑指南（葛军错题本）

### ❌ 反模式

**1. 炫技型**
- 错误：为了展示AI能力而做复杂功能
- 纠正：功能服务于真实需求，而非相反

**2. 全或无型**
- 错误：只有全自动和全手动两种模式
- 纠正：设计"半自动"过渡态，人机协作

**3. 失忆型**
- 错误：每次对话从零开始
- 纠正：Memory Skill，记住用户偏好和历史

**4. 黑盒型**
- 错误：AI直接给结果，不解释过程
- 纠正：展示思考过程，让用户理解"为什么这么建议"

## 四、检验清单：你的技能是"好题"吗？

发布前问自己：

```
□ 1. 真实场景测试：找3个真实用户用过，他们真的觉得有用吗？
□ 2. 边界测试：用户输错/网络断/平台改版时， gracefully fail 了吗？
□ 3. 认知负荷：用户需要学新东西吗？还是符合既有习惯？
□ 4. 惊喜测试：有没有一个点，让用户觉得"哇，这都想到了"？
□ 5. 长期测试：用10次后，是更顺手了还是更烦了？
```

## 五、从出题到出Skill：能力迁移表

| 数学命题能力 | AI技能设计能力 |
|-------------|---------------|
| 理解数学思想本质 | 理解用户需求本质 |
| 设计认知脚手架 | 设计交互引导流程 |
| 控制难度梯度 | 设计MVP到高级功能阶梯 |
| 埋设思维锚点 | 植入AI主动建议时机 |
| 制造顿悟时刻 | 设计超预期惊喜 |
| 反套路创新 | 不抄竞品，解决真问题 |

## 六、实战：用方法论重做小红书Skill

**原设计（炫技导向）：**
```
功能：自动登录 → 填标题 → 填内容 → 点发布 → 结束
问题：平台一改版就崩溃，用户不知道怎么救
```

**葛军式重构（问题导向）：**
```
核心问题：IP主理人每天花1小时发小红书，想压缩到10分钟

Level 1 - 能跑：
  - 一键发布（含容错：验证码弹窗提醒用户手动处理）
  
Level 2 - 能省：
  - 草稿箱自动同步（本地改完一键同步）
  - 最佳时间提醒（基于历史数据分析）
  
Level 3 - 能优：
  - 内容模板库（高频场景一键套用）
  - 数据复盘（发布后自动总结效果）
  
Level 4 - 能想：
  - 热点监控 → 主动建议选题
  - 竞品分析 → 建议差异化内容
  
锚点设计：
  - 每次发布前问："今天是追热点还是日常？"
  - 根据选择，自动调整发布策略
  
惊喜时刻：
  - "你连续7天8点发布，互动率比随机时间高42%，建议继续保持！"
```

## 结语

**葛军说**："好的数学题，是让学生做完后觉得数学有用、有趣、有美。"

**树大虾说**："好的AI技能，是让用户用完后觉得AI有用、懂我、像伙伴。"

方法论是通用的，区别在于：
- 数学命题面对的是**认知规律**
- AI技能设计面对的是**人性需求**

但都遵循：**真实 → 阶梯 → 锚点 → 惊喜**
